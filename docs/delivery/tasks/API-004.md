# API-004 — HTTP Security and Lifecycle Routes

**Implementer inference:** Low  
**Prerequisites:** `API-001`, `API-005`, and `API-003` approved.

## Frozen inputs

Use the exact routes, headers, auth scopes, bodies, state enums, retry semantics, error codes, origins, limits, and manifests in `CONTRACTS.md` and every deadline in `TIMEOUTS.md`. Replace only API-001 HTTP/status stubs. Stop if a shared build, dependency, entrypoint, or aggregate-test file must change.

## Objective

Implement the frozen REST surface, authentication scopes, cached component status, retries, and shutdown.

## Procedure

1. Implement only the routes and response schemas in `CONTRACTS.md`.
2. Enforce exact-origin CORS without credentials, 64 KiB request limits, `Authorization: Bearer`, `X-Request-Id` on every response, and constant-time token comparison.
3. Keep liveness unauthenticated and process-local; readiness reflects component readiness.
4. Cache background probe state without overlapping component probes.
5. Poll at one second while transitioning/unhealthy and ten seconds while healthy.
6. Make retry idempotence explicit: return `202`, suppress duplicate active retries, and never restart the healthy component.
7. Restrict shutdown to the supervisor token, quiesce new requests, protect
   databases/logs/owned resources, and begin orderly asynchronous process
   shutdown. The pinned api.zig accept loop is not required to return.
8. Return only the frozen statuses, bodies, and sanitized error envelopes/codes from `CONTRACTS.md`.
9. Return `503 native_shutdown_timeout` only if a native deinit watchdog failure
   is observable before mandatory exit `7`.

- Missing, malformed, wrong-scope, and correct-token tests.

- Golden tests cover every frozen response body, state value, and error code; `zig build test` and `git diff --check` pass.
