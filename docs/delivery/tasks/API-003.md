# API-003 — TigerBeetle C Client Adapter

**Implementer inference:** Low  
**Prerequisites:** `API-001` and `FEAS-002` approved.

## Frozen inputs

Use `DEPENDENCY-PINS.md`, `CONTRACTS.md`, and `TIMEOUTS.md`. Replace only the API-001 TigerBeetle stub. Stop if a shared build, dependency, entrypoint, or aggregate-test file must change.

## Objective

Convert the approved C ABI spike into a production-quality Zig adapter.

## Procedure

1. Wrap client creation, address parsing, submission, callback completion,
   five-second request timeout, and cleanup.
2. Keep every C pointer, context, and buffer alive until its callback finishes.
3. Serialize or coordinate operations according to the native client contract.
4. Implement a harmless real lookup health probe.
5. Map native failures to stable sanitized internal codes.
6. Call official unmodified `tb_client_deinit` on a dedicated shutdown thread
   with a ten-second watchdog. Join it when it returns.
7. If deinit misses ten seconds, terminate the API process immediately with
   exit code `7`; do not detach work or allow it to outlive process exit.

## Acceptance evidence

- Success, unavailable-server, timeout, callback, cancellation/shutdown, and
  repeated-init tests.
- Real deinit-under-ten-seconds evidence plus an injectable stalled-deinit child
  fixture whose parent observes exit `7` within 12 seconds and no survivor.
- Native leak/sanitizer evidence where available.
- Error-redaction scan.
- `zig build test` succeeds through API-001's static aggregate registration; `git diff --check` passes.
- Native adapter acceptance is Windows 11 x64 only; optional non-Windows
  compilation is informational/non-support.

## Reviewer focus

Audit pointer and callback lifetimes, races, double completion, dedicated-thread
shutdown ordering, mandatory exit-7 containment, and proof that no CLI or proxy
is used.
