# CI-001 — Non-Deployment GitHub Actions

**Implementer inference:** Low  
**Prerequisites:** `PACKAGE-001` approved.

## Frozen inputs

Use toolchain/dependency pins from `DEPENDENCY-PINS.md`, protocol contracts from `CONTRACTS.md`, exact job/retention limits from `TIMEOUTS.md`, and artifact names/layouts/provenance from `PACKAGING.md`.

## Objective

Automate verification and short-lived smoke artifacts without restoring cloud deployment scope.

## Procedure

1. Add static/unit, Compose, managed smoke, Windows ZIP smoke, and security jobs
   whose current native gate is Windows 11 x64.
2. Call reviewed repository scripts; do not duplicate lifecycle logic in workflow YAML.
3. Pin every action to a full commit SHA.
4. Use read-only permissions, no repository secrets, 30-minute ordinary job timeouts, 90-minute native build/package job timeouts, and least artifact access.
5. Upload sanitized logs only on failure and retain artifacts exactly seven days.
6. Configure Dependabot for relevant ecosystems with no automerge.
7. Ensure no Terraform, AWS, deployment, publishing, signing, or notarization step exists.

## Acceptance evidence

- Workflow syntax validation and job-to-script map.
- Action SHA and permission audit.
- Successful matrix run links.
- Windows-only current-gate inventory; any non-Windows compile experiment is
  non-gating and labeled non-native/non-support.
- Artifact/log retention and secret-usage inventory.
- Workflow syntax command, action-SHA audit command, permissions search, timeout/retention search, and `git diff --check`.

## Reviewer focus

Audit provenance, permissions, Windows gate coverage, timeout/cleanup behavior,
and every workflow for deployment creep or unsupported-platform claims.
