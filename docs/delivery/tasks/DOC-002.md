# DOC-002 — Verified Development and Operations Runbooks

**Implementer inference:** Low  
**Prerequisites:** `CI-001`, `DBDOC-001`, and `DOC-001` approved.

## Frozen inputs

Use `DEPENDENCY-PINS.md`, `CONTRACTS.md`, `TIMEOUTS.md`, and `PACKAGING.md` as the exact source for versions, commands, behavior, timeouts, names, layouts, limitations, and provenance. Do not document alternatives or aspirational behavior.

## Objective

Document only commands and behavior that have been independently exercised.

## Procedure

1. Write setup instructions for Windows 11 x64, macOS Intel/Apple Silicon, and Ubuntu 24.04 x64.
2. Explain bootstrap profiles, environment doctor, Compose external mode, desktop managed mode, and all test commands.
3. Explain process ownership, tokens, ports, logs, status states, retries, safe shutdown, and troubleshooting.
4. Document portable artifact execution, macOS ad-hoc-sealed/unnotarized limitations, and the absence of production signing on every platform.
5. Document stopped-application directory-copy backups.
6. State clearly that reset, repair, upgrade, automatic backup, installers, and auto-update are absent.
7. Explain module registration and ADR supersession for future work.
8. Have an independent operator execute every documented command.

## Acceptance evidence

- Platform-by-platform command verification table.
- Links to successful outputs or corrected discrepancies.
- Link and terminology checks.
- Exact command-verification output locations for every documented command plus `git diff --check`.

## Reviewer focus

Follow the guides from a clean environment, reject undocumented prerequisites or aspirational commands, and return every mismatch to the writer.
